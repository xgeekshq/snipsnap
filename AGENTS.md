# snipsnap - AI Agent Guide

Aggressive single-GPU LLM switching operator for Kubernetes. Only one model is loaded at a time; requesting a different model kills the current pod and spins up a new one. Targets edge/homelab environments (e.g. a single RTX 4090).

## Architecture

Two CRDs work together:

- **Model** -- Declarative config: URL, engine (OLlama/VLLM), cache settings, resources. Does not run anything by itself.
- **Workspace** -- State machine controlling the GPU. Has an `activeModel` field and tracks phase: `Idle → Switching → Loading → Ready`.

The **proxy** (OpenAI-compatible API on `:8000`) receives inference requests, extracts the model name from the JSON body, patches the Workspace if a switch is needed, polls until ready, then reverse-proxies to the inference pod.

API group: `snipsnap.xgeeks.com/v1`

## Labels and Annotations

Defined in `api/v1/labels.go`:

| Constant | Value | Used on |
|----------|-------|---------|
| `LabelManagedBy` | `app.kubernetes.io/managed-by` | All owned resources |
| `LabelModel` | `xgeeks.com/model` | Pods, PVCs |
| `LabelWorkspace` | `xgeeks.com/workspace` | Pods |
| `LabelEngine` | `xgeeks.com/engine` | Pods |
| `ModelPodPortAnnotation` | `xgeeks.com/port` | Pods (inference port) |
